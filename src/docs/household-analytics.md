# Suka Smart Assistant — Household Analytics

**File:** `src/docs/household-analytics.md`  
**Purpose:** Define the analytics layer that sits *on top of* the SSA event stream and household data tables. This layer turns:
- imports → signals
- inventory / storehouse → shortages & provisioning plans
- sessions → execution & compliance
- garden / animal / preservation → production vs. consumption
- (optional) hub exports → community insight

…into **metrics and formulas** that the UI (e.g. `src/analytics/HouseholdAnalytics.jsx`) can render.

This document explains:
1. **Data sources** (which Dexie tables / events to read)
2. **Metrics** (names, formulas, inputs)
3. **Dimensions** (time, domain, household, member)
4. **Event → metric mapping**
5. **Forward-thinking extension points** (preservation, animal, storehouse)

---

## 1. Data sources

SSA is **event-driven**, so most analytics can be calculated either:

1. **Streaming** — update counters when an event is emitted, **or**
2. **Batch** — read Dexie tables and aggregate

You have all of these already in your schema:

- `events`
  - event bus mirror — every `{ type, ts, source, data }`
  - **primary source** for time-based analytics

- `imports`
  - canonical, domain-aware imports
  - includes `contextIntelligence` → good for content mining

- `inventory`
  - actual, current stock
  - has thresholds → source for shortages

- `storehouseGoals`
  - desired stock → source for *gaps*

- `sessions`
  - actionable work generated by automation or user
  - track created vs executed vs missed

- `gardenHarvests`
  - real production events

- `animalAssets`, `butcherySessions`
  - animal capacity + real outputs

- `preservationBatches`
  - conversion of raw → shelf-stable

---

## 2. Core analytics questions

SSA analytics should answer:

1. **What’s coming in?**  
   - imports per domain
   - successful vs failed imports
   - content → session conversion

2. **What are we doing?**  
   - sessions created vs completed
   - domain mix (meals vs cleaning vs garden vs animal vs preservation)

3. **What do we have?**  
   - inventory value/levels
   - storehouse goal attainment
   - shortages & recurring shortages

4. **What are we producing?**  
   - garden harvest amounts
   - butchery outputs
   - preserved goods

5. **What should the Hub see?**  
   - exportable events
   - household production potential
   - demand signals from shortages

## 3. Metric catalog

Below is a **starter** metric set. You can implement each as a function in `src/analytics/HouseholdAnalytics.jsx` or a separate service.

### 3.1 Imports

**Metric:** `imports.total`  
- **Description:** total number of normalized imports  
- **Source:** `imports` table  
- **Formula:**  
  ```text
  imports.total = COUNT(imports)
Metric: imports.byDomain

Description: imports grouped by domain

Source: imports.domain

Formula:

text
Copy code
imports.byDomain[d] = COUNT(imports WHERE domain = d)
where d ∈ { recipe, cleaning, garden, animal, storehouse, video, preservation? }

Metric: imports.successRate

Description: success vs error

Sources: imports, importErrors

Formula:

text
Copy code
imports.successRate = COUNT(imports) / (COUNT(imports) + COUNT(importErrors))
Metric: imports.toSessionConversion

Description: how many imports actually produced “work”

Sources: importToSessionRequests, sessions

Formula (simple):

text
Copy code
imports.toSessionConversion = COUNT(importToSessionRequests) / COUNT(imports)
Formula (strict): if you want only those requests that actually produced a session:

text
Copy code
imports.toSessionConversionStrict =
    COUNT(sessions WHERE origin = 'import') / COUNT(imports)
3.2 Sessions
Metric: sessions.created

Description: number of actionable sessions created

Source: sessions table or session.created events

Formula:

text
Copy code
sessions.created = COUNT(sessions)
Metric: sessions.byType

Description: distribution of work

Source: sessions.type

Formula:

text
Copy code
sessions.byType[t] = COUNT(sessions WHERE type = t)
where t ∈ { cooking, cleaning, garden, animal, preservation, storehouse }

Metric: sessions.completionRate

Description: % of scheduled sessions that were actually done

Prereq: you need to emit or mark status = 'completed' and status = 'missed'

Formula:

text
Copy code
sessions.completionRate =
    COUNT(sessions WHERE status = 'completed')
    /
    COUNT(sessions WHERE status IN ('completed', 'missed', 'canceled'))
Metric: sessions.fromImportsShare

Description: how much of household work was suggested by imports

Formula:

text
Copy code
sessions.fromImportsShare =
    COUNT(sessions WHERE origin = 'import')
    /
    COUNT(sessions)
3.3 Inventory
Metric: inventory.items

Formula:

text
Copy code
inventory.items = COUNT(inventory)
Metric: inventory.itemsBelowThreshold

Description: how many items are under minThreshold

Formula:

text
Copy code
inventory.itemsBelowThreshold =
    COUNT(inventory WHERE quantity < minThreshold)
Metric: inventory.shortagesPerWeek

Description: turn event stream into rate

Source: events table, filter by type = 'inventory.shortage.detected'

Formula (last 7 days):

text
Copy code
inventory.shortagesPerWeek =
    COUNT(events WHERE type = 'inventory.shortage.detected'
                  AND ts >= NOW - 7d)
Metric: inventory.torahCleanShare

Description: in your system you track clean vs unclean

Formula:

text
Copy code
inventory.torahCleanShare =
    COUNT(inventory WHERE isTorahClean = true)
    /
    COUNT(inventory)
3.4 Storehouse
Metric: storehouse.goals

Formula:

text
Copy code
storehouse.goals = COUNT(storehouseGoals)
Metric: storehouse.goalsByPriority

Formula:

text
Copy code
storehouse.goalsByPriority[p] = COUNT(storehouseGoals WHERE priority = p)
Metric: storehouse.attainment

Description: how close the household is to its target stock

Needs: link between storehouseGoals and inventory (we created fields like linkedInventoryId)

Formula (per goal):

text
Copy code
attainment_i =
  ( currentQty(inventory, goal.linkedInventoryId) / goal.targetQuantity ) * 100
Formula (household-level):

text
Copy code
storehouse.attainment =
    AVG(attainment_i for all goals)
This is a key Hub signal: if many households have low attainment on “flour,” that’s what Hub grocery stores should stock.

4. Time dimensions
Most metrics should support at least:

Today

Last 7 days

Last 30 days

This calendar month

This scriptural / Hebrew month (for your calendar module)

YTD

Since events have ISO timestamps, filtering is simple:

js
Copy code
events.filter(e => e.ts >= START && e.ts <= END)
You can codify this in a helper: getEventsByRange(db, { start, end, type? }).

5. Domain dimensions
Everything in SSA is multi-domain by design. That means every metric should be filterable by domain:

recipe

cleaning

garden

animal

storehouse

video

preservation

Example:
“imports in the last 30 days for garden”

text
Copy code
imports.byDomain['garden', last30d] =
    COUNT(imports WHERE domain = 'garden' AND createdAt >= NOW - 30d)

Event type	Metrics affected
import.parsed	imports.total, imports.byDomain, imports.successRate
import.session.generate.requested	imports.toSessionConversion
inventory.updated	inventory.items, inventory.itemsBelowThreshold
inventory.shortage.detected	inventory.shortagesPerWeek
storehouse.goal.created	storehouse.goals, storehouse.goalsByPriority
garden.plan.created	garden.plans
garden.harvest.logged	garden.harvestsPerMonth, garden.totalHarvestedWeight
animals.asset.created	animals.activeAssets
animals.butchery.session.created	animals.butcherySessions
storehouse.stock.fromButchery	animals.butcheryToStorehouseRate, storehouse.attainment
preservation.completed	preservation.batches, preservation.outputsTotal
session.created	sessions.created, sessions.byType

7. Implementation notes (forward-thinking)
Keep formulas as pure functions.
Example (src/analytics/HouseholdAnalytics.jsx):

js
Copy code
export function calcImportsTotal(imports = []) {
  return Array.isArray(imports) ? imports.length : 0;
}
Add an in-memory aggregator that listens to window.__suka?.eventBus and “increments” counters without re-querying Dexie each time.

Support new domains easily:

if you add preservation imports later, they will still emit import.parsed → all import metrics still work

you just add preservation.* to sessions.byType and imports.byDomain

Do not break offline.
Analytics should work even if Hub is down — you only add extra fields when exports succeed.

9. Summary
Imports give you intent and content.

Inventory + storehouse give you need and gap.

Sessions give you execution.

Garden / Animals / Preservation give you production.

Hub export gives the community what to make and when.

This doc defines the analytics vocabulary so the rest of SSA can talk about the same numbers. ✅
